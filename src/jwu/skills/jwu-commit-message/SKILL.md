# jwu: коммит-сообщение из текущих изменений

## Когда

Во время цикла работы (job) после правок — нужен текст коммита по тому, что сейчас
накопилось в рабочем дереве. Скилл **показывает текст**, `git add`/`git commit`/`git push`
делает пользователь сам (правило проекта).

## Что считается «изменениями»

Все текущие изменения относительно `HEAD` — staged + unstaged + новые отслеживаемые
файлы. Не имеет значения, что застейджено, а что нет: формируем сообщение по всей
сумме правок.

Команда для сбора: `git diff HEAD` (покажет и staged, и unstaged; для статуса
по файлам — `git diff HEAD --stat`). Новые untracked файлы тоже учитываем:
`git status --short`.

## Формат

С префиксом задачи (если он есть в имени ветки):

```
<НОМЕР_ЗАДАЧИ>: Краткое описание

- что было сделано (10–15 слов)
- ещё пункт
```

Без префикса (ветка `main`/`develop`/без префикса в имени):

```
Краткое описание

- …
```

## Алгоритм

1. **Ветка:** `git rev-parse --abbrev-ref HEAD`.
2. **Префикс задачи** — извлечь из имени ветки регуляркой `^[A-Z][A-Z0-9]+-[0-9]+`
   (напр. `EXAMPLE-1111-foo-bar` → `EXAMPLE-1111`, `WEBIMCORE-12507-x` → `WEBIMCORE-12507`).
   - Не нашли? **НЕ выдумывать.** Печатать сообщение без префикса и предупредить пользователя.
3. **Что менялось** — собрать всё:
   - `git diff HEAD --stat` — обзор по файлам.
   - `git diff HEAD` — содержимое правок.
   - `git status --short` — добавит новые untracked файлы (строки `??`).
   - Изменений нет вообще (всё чисто) — сообщи «нет изменений», ничего не печатай.
4. **Язык тела** — определить по `git log -10 --pretty=%s` и писать на этом языке
   (в большинстве jwu-репозиториев — русский).
5. **Заголовок** — одна строка, суть изменений:
   - С префиксом: `<НОМЕР>: <краткое описание>`.
   - Без префикса: просто `<краткое описание>`.
6. **Список** — `- ` пунктами по 10–15 слов: что конкретно сделано (только из реального
   diff/status, не из памяти разговора). Один пункт = одно изменение или группа.
7. **Вывести готовый текст в код-блоке.** Никаких `git add`/`git commit`.

## Пример

Ветка `EXAMPLE-1111-fix-cookie-name`, в дереве (staged + unstaged) — правка cookie-имени:

```
EXAMPLE-1111: Имя токена авторизации вынесено в .env

- Добавлена переменная окружения для имени cookie авторизации
- Поправлено форматирование URL на странице логина
```

Ветка `main`, в дереве — мелкая правка README:

```
Уточнение в README по установке скиллов

- Добавлено пояснение про флаг --skip-agents
```

(+ предупреждение: «Префикс задачи не найден в ветке `main`, пишу без префикса».)

## Частые ошибки

| Ошибка | Правильно |
|---|---|
| Смотрим только staged или только unstaged | Брать всё через `git diff HEAD` + `git status --short` |
| Выдумываем номер задачи | Только из ветки регуляркой; нет — без префикса + предупреждение |
| Пишем пункты «по памяти разговора» | Только из реального `git diff HEAD` |
| Сами вызываем `git add`/`git commit` | Только печатаем текст; коммитит пользователь |
| Добавляем `Co-Authored-By` / упоминание AI | НИКОГДА, никакой атрибуции |
